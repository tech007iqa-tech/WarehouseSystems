# Sitemap & Directory Structure

## 1. Directory Structure
This is the required folder tree for the application. Any new files created by developers or AI agents MUST adhere to this structure to maintain organization and "vibe".

```text
/LabelAPP/
│
├── /assets/                # Static frontend assets
│   ├── /css/
│   │   └── style.css       # The single, global stylesheet (No Bootstrap/Tailwind)
│   ├── /js/
│   │   ├── main.js         # Global JS (Nav toggles, theme switching)
│   │   ├── forms.js        # Form validation and dynamic dependencies
│   │   └── api.js          # Fetch API wrappers for database calls
│   └── /icons/             # Local SVG icons (No external CDNs)
│
├── /db/                    # SQLite Database Files
│   ├── labels.sqlite       # (Generated by db.php)
│   ├── orders.sqlite       # (Generated by db.php)
│   └── rolodex.sqlite      # (Generated by db.php)
│
├── /templates/             # Master files for PowerShell Injection
│   ├── label_template.odt  # The raw ODT format for hardware labels
│   ├── order_template.ots  # The raw OTS format for B2B Purchase Forms
│   └── /scripts/           
│       ├── generate_odt.ps1  # PowerShell script to inject XML into ODT
│       └── generate_ots.ps1  # PowerShell script to inject XML into OTS
│
├── /includes/              # Reusable PHP backend components
│   ├── db.php              # Initializes PDO connections to the 3 SQLite files
│   ├── header.php          # The opening HTML, `<head>`, and Sidebar Nav
│   ├── footer.php          # The closing HTML and script tags
│   └── functions.php       # Core PHP helpers (sanitization, formatting)
│
├── /api/                   # API Endpoints (Returns JSON for JS fetch calls)
│   ├── labels_api.php      # CREATE/READ/UPDATE/DELETE hardware labels
│   ├── orders_api.php      # Generate Purchase Forms & log orders
│   └── rolodex_api.php     # Manage customer/lead data
│
├── /exports/               # User-generated files (Gitignored)
│   ├── /labels/            # Where generated .odt files are saved for download
│   └── /orders/            # Where generated .ots files are saved for download
│
├── index.php               # Dashboard (Stats, Quick Search)
├── labels.php              # Hardware Inventory List View (Search/Filter warehouse)
├── new_label.php           # Form to create a new physical label (.odt)
├── orders.php              # Purchase Orders List View
├── new_order.php           # Form to basket items into an order (.ots)
├── rolodex.php             # Customer & Leads List View
├── new_customer.php        # Form to add a new lead/customer
│
├── README.md               # Project Context
├── ARCHITECTURE.md         # Database Schemas & Logic
├── SITEMAP.md              # (This file) Directory Structure
├── DESIGN_SYSTEM.md        # CSS Variables and Layout Rules
└── ROADMAP.md              # Build Phases
```

## 2. UI View Hierarchy (The Sitemap)

The application is built around a persistent Sidebar Navigation (`includes/header.php`) that links to the following primary views:

### Dashboard (`index.php`)
* **Purpose:** The home screen. Provides high-level warehouse statistics.
* **Key Elements:**
    * Total Inventory Count (Labels `status = 'In Warehouse'`)
    * Recent Orders Table (Last 5 orders)
    * Active Leads Count
    * Quick Barcode Scanner / Search Input.

### 📦 Hardware Inventory
* **`labels.php` (Warehouse List):** A data table displaying all hardware. Includes filters for Location (In Warehouse vs. Sold), Brand, Model, and Condition.
* **`new_label.php` (Create Entry):** The dynamic form where users input RAM/Storage/CPU specs, assign a warehouse bin, and click "Generate Label" to print the `.odt`.

### 🧾 Purchase Orders
* **`orders.php` (Order History):** A list of all historical B2B orders. Allows downloading past `.ots` files.
* **`new_order.php` (Create Order):** A specialized interface where users can select a Customer from the Rolodex, search the Warehouse for available hardware (`labels.sqlite`), add items to an order basket, specify the Quantity/Price, and generate the final `.ots` B2B form.

### 📇 Customer Rolodex
* **`rolodex.php` (CRM List):** A table displaying all Leads and Active Customers. Clicking a row expands their details and order history.
* **`new_customer.php` (Add Lead):** A quick form to add Name, Company, Phone, Email, and internal Notes.

## 3. The "API First" Strategy
When building the UI views (e.g., `labels.php`), the PHP files should primarily just render the HTML skeleton. 

The actual data fetching, form submissions, and database writing should be handled by **Vanilla JavaScript** in `/assets/js/` communicating with the endpoints in our `/api/` directory via `fetch()`. This prevents clunky full-page reloads and makes the application feel like modern, premium software.
